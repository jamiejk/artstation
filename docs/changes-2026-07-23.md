# July 2026 operator and gradual-pen update

This document inventories the work committed after `9a2b72c` (“Improve
auto-dip timing and server organization”). It groups the changes by operator
behavior rather than by file.

## Gradual pen motion and calibration

- Added a vendored, opt-in AxiDraw 3.9.6 fork that can begin raising or lowering
  the servo while upstream-planned XY motion is still running.
- Retained the first `soft_out_mm` proof and added a multi-step gradual profile
  with independent entry distance, exit distance, fully raised tail, and
  target spacing.
- Added Standard and Staedtler Marsmatic profiles. Profile parameters are
  snapshotted into jobs so queued work is reproducible.
- Selected the Marsmatic production values from physical strips: 4.375 mm
  lowering, 4.6875 mm raising, 0.4375 mm raised tail, and roughly 0.5 mm target
  spacing.
- Added live Z jogging, capture-current-height controls, absolute direct-EBB
  servo positioning, a contact/clearance calibration routine, and safer
  standard-servo PWM setup.
- Expanded the probed local servo map while retaining comments about physical
  stops and safe incremental testing.
- Added manual pen-delay and Z-ramp experiments plus retained SVG proof/sweep
  fixtures.

The fork's provenance, isolation boundary, changed upstream files, invariants,
tests, and update procedure are documented in
[`../vendor/axidraw-softout/README.md`](../vendor/axidraw-softout/README.md).

## Pause and job recovery

- Added a dedicated pause-clearance height. After AxiDraw has safely paused and
  released serial ownership, the server raises to that clearance without ever
  lowering a pen that is already higher.
- Centralized paused-job finalization so initial runs, resumed runs, and
  dip-recovery runs use the same clearance behavior.
- Added Continue and Restart-from-beginning controls directly to paused job
  cards.
- Added cancellation and 90-degree rotation controls to queued job cards.
- Preserved failed and completed work in Recent Jobs, with Reload creating a
  new job from the original inputs.

## Upload, preview, and rotation

- File selection now immediately loads, queues, and previews the chosen SVGs.
- Added quarter-turn rotation (`0`, `90`, `180`, `270`) before dimension,
  preview, paper-fit, and keep-out analysis.
- Added rotation-aware reruns and queued-job replacement.
- Kept unanchored SVG previews visible when bed calibration is unavailable,
  explicitly labelling them as shape-only previews.

## Position and hardware safety

- Made position trust explicit. Switching motors off invalidates the plotted
  cursor and Home because the carriage can move independently afterward.
- Hides the live cursor while AxiCLI owns the plotter and reconciles it from EBB
  step counters after AxiCLI exits.
- Invalidates stale coordinates when reconciliation fails or a restart occurs
  during motion.
- Clarified calibrated absolute movement versus calibration-off local relative
  jogging. Calibration-off Home establishes a local `(0,0)` but does not claim
  a bed coordinate.
- Bound ink-well calibration/test state to the bed-calibration identity so
  stale physical coordinates cannot silently become valid after recalibration.

## Control-page workflow

- Reworked status modals and control enablement to explain motor, calibration,
  position-owner, and preview state transitions.
- Added clearer pen-profile, Z-height, pause-clearance, timing, rotation, and
  job-detail displays.
- Preserved Space as the plot pause/continue shortcut while preventing numeric
  inputs from swallowing the operator action unexpectedly.

## Desktop launcher

- Added `scripts/launch_plotter_gui.sh`, which starts the user systemd service,
  waits for `GET /control` to succeed, and opens the control page.
- Added `launch-plotter.desktop` as the installable Desktop entry used by the
  local **Launch Plotter** button.

## Verification

The automated suite covers server policy, hardware command construction,
pause finalization, SVG rotation, job snapshots, profile selection, AxiCLI
command construction, the fork transforms, and the public wrapper options.
Hardware experiments remain under `manual_tests/` and are never part of
automatic discovery.

Run:

```bash
PLOTTER_DISABLE_WORKER=1 venv/bin/python -m unittest discover -v
```
